## MongoDB — CP и AP

По умолчанию MongoDB обеспечивает строгую консистентность (с 5ой версии). 
Чтение и запись идут через Primary, используются `readConcern: "majority"` 
и `writeConcern: "majority"`. Поэтому данные считаются 
записанными только когда их подтвердило большинство узлов.

При сплите сети (partition) только та часть кластера, 
где есть большинство голосов, может выбрать Primary и продолжать 
принимать записи. Узлы в меньшинстве перестают обрабатывать запросы. 
Здесь Availability приносится в жертву консистентности.

MongoDB можно настроить так, чтобы пожертвовать консистентностью ради доступности:

```javascript
// Запись — подтверждение только от Primary, не ждём репликации
db.collection.insertOne(
  { name: "test" },
  { writeConcern: { w: 1 } }
)

// Чтение — с ближайшего узла, даже если данные устарели
db.collection.find({}).readPref("nearest")

// Или чтение с вторичных узлов
db.collection.find({}).readPref("secondary")

// Читать локальные данные без ожидания majority
db.collection.find({}).readConcern("local")
```

При такой конфигурации: Primary продолжает принимать записи при сплите, 
клиенты читают с любого доступного узла. 
Данные могут быть несогласованными между узлами, но система остаётся 
доступной.

## MSSQL — CP

MSSQL — классическая реляционная СУБД с полной поддержкой ACID. Строгая консистентность — поведение по умолчанию.

В распределённой конфигурации при потере связи между узлами MSSQL предпочитает консистентность. 
Без кворума (majority) база переходит в состояние, недоступное для записи, 
пока не будет выбран новый Primary или не восстановится связь.

## Cassandra — AP

Cassandra изначально проектировалась как система с максимальной доступностью. Архитектура peer-to-peer — нет единого мастера, все узлы равноправны.

При сплите сети каждый узел продолжает принимать и чтение, и запись. 
Конфликты при записи разрешаются по принципу last write wins - побеждает запись 
с более поздним timestamp. Результат - eventual consistency: данные на 
узлах со временем сходятся, но в конкретный момент могут отличаться.

Consistency level можно настроить на уровне запроса (ONE, QUORUM, ALL и другие промежуточные).
